When locating code, symbols, text, files, references, or C/C++ header/source counterparts, prefer the ace-code-search-mcp Skill and its indexed MCP tools when a matching Ace Code Search index exists. Use list_indexes to select the workspace index, search_code for indexed search, read_indexed_file for indexed snapshots, and find_header_source for indexed C/C++ pairing. Fall back to rg, filesystem search, or direct reads when no matching index exists, partialIndex is true and completeness matters, results may be stale, or the target is excluded or unindexed. Do not claim indexed results include unsaved or not-yet-indexed changes.
